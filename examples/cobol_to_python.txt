Migrate Payroll System from COBOL to Python

## Source

The source codebase is a COBOL weekly payroll calculator located in the `src/` folder.
The main program is `PAYROLL.cbl`.
The input data file format is `employees.dat`.

## Target

Migrate to **Python 3.12**. Produce idiomatic, well-structured Python code.

## Requirements

- Preserve all business logic exactly:
  - Regular hours: first 40 h at base hourly rate
  - Overtime hours: hours beyond 40 at 1.5x the base rate
  - Tax bracket 1: gross pay <= $500 -> 10% tax
  - Tax bracket 2: gross pay $501-$1500 -> 20% tax
  - Tax bracket 3: gross pay > $1500 -> 30% tax

- Replace COBOL flat-file I/O with Python file reading. The input file
  (`employees.dat`) is a fixed-width record file where each line is:
    - Columns 1-5:   Employee ID (5 chars, zero-padded numeric)
    - Columns 6-35:  Employee name (30 chars, space-padded)
    - Columns 36-39: Hours worked (4 chars, NNNN, last digit is tenths; e.g. 0450 = 45.0 hours, 0425 = 42.5 hours)
    - Columns 40-45: Hourly rate  (6 chars, NNNNNN, last 2 digits are cents; e.g. 001500 = $15.00, 025000 = $250.00, 122050 = $1220.50)

  Parse hourly rate as: integer(columns 40-45) / 100 = dollar amount.
  The system supports rates up to $9,999.99/hour to cover the full range from hourly staff to senior contractors.

- Produce a formatted payroll report to stdout (or a file), matching the
  structure of the COBOL REPORT-FILE output: header, one detail line per
  employee showing ID, name, gross, tax and net pay, then a summary line
  with totals and employee count. The report must match this layout exactly:

  ```
  WEEKLY PAYROLL REPORT
  =======================================================================
  EMP   NAME                           GROSS PAY    TAX          NET PAY
  ID
  ----- ------------------------------ ------------ ------------ ------------
  12345 JOHN SMITH                      $    750.00 $     75.00  $    675.00
  67890 JANE DOE                        $   2100.00 $    630.00  $   1470.00
  ----- ------------------------------ ------------ ------------ ------------
  TOTAL EMPLOYEES: 2                    $   2850.00 $    705.00  $   2145.00
  ```

  Column widths: ID 5, name 30, gross/tax/net each 12 (right-aligned, with $ prefix).
  Separator lines use dashes matching each column width.

- Use dataclasses or a named tuple to represent an employee record.
- Separate concerns into distinct functions or classes matching the COBOL paragraph structure:
  - Parsing / reading input records
  - Calculating gross pay (regular + overtime)
  - Calculating tax
  - Accumulating totals
  - Formatting and printing the report

- Include comprehensive unit tests using pytest covering:
  - Regular hours calculation (no overtime)
  - Overtime hours calculation
  - All three tax brackets (boundary values)
  - Full end-to-end run with the provided `employees.dat`

## Constraints

- Use only the Python standard library (no third-party packages).
- Monetary values must use Python `decimal.Decimal` to avoid floating-point errors.
- The output report must be human-readable and match the column layout of the original COBOL report.
